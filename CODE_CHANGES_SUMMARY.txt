PROBLEM SOLVED
═══════════════════════════════════════════════════════════════════════════════

Before Fix:
  1. App starts
  2. Tries to register device immediately
  3. NO API key in request
  4. Backend returns error: "Missing x-api-key header"
  5. App shows: "Connection Failed"
  ❌ Application unusable

After Fix:
  1. App starts
  2. Checks for API key
  3. If missing → Show API Key Setup screen (NEW)
  4. User enters API key from admin dashboard
  5. Save to device
  6. Device registration includes API key
  7. Backend validates API key
  8. ✅ Connection succeeds
  9. App ready to use

HOW IT WORKS: DATA FLOW
═══════════════════════════════════════════════════════════════════════════════

User Journey:

1. FIRST TIME USER
   App Launch
   ↓ (index.tsx checks AsyncStorage)
   API Key Not Found
   ↓
   Show: API Key Setup Screen (api-key-setup.tsx) ← NEW
   ↓ (User action)
   User enters: "scanner_xxx_xxx"
   ↓
   Validated: Starts with "scanner_"?
   ↓
   Save: DeviceService.setApiKey(apiKey)
   ↓ (Stored in AsyncStorage)
   Saved to device!
   ↓
   Navigate: To device-setup

2. DEVICE SETUP
   Show: Device Setup Screen
   ↓ (User action)
   User enters: "Gate 1"
   ↓
   Register: DeviceService.registerDevice()
   ├─ Get API key: DeviceService.getApiKey()
   ├─ Add header: {'x-api-key': apiKey}
   ├─ Send request: POST /api/scanner/register-device
   └─ Backend validates API key ✓
   ↓
   Registered!
   ↓
   Navigate: To scanner

3. SUBSEQUENT LAUNCHES
   App Launch
   ↓ (index.tsx checks AsyncStorage)
   API Key Found!
   ↓ (Skip api-key-setup)
   Check Device Configured?
   ├─ If No → Device Setup
   └─ If Yes → Scanner Ready

  AsyncStorage
  ├─ device_config: {...}
  ├─ api_key: "scanner_xxx_xxx" ← NEW
  ├─ cached_users: [...]
  ├─ scan_history: [...]
  ├─ api_server: "SOLESTA"
  └─ last_sync: "2026-04-16T12:00:00Z"
