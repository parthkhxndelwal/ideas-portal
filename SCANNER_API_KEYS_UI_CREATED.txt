───────────────────────────────────────────────────────────────────────────────
HOW TO GENERATE A SCANNER API KEY (NOW AVAILABLE!)
───────────────────────────────────────────────────────────────────────────────

Step 1: Log in to Admin Dashboard
  URL: https://solesta.krmangalam.edu.in/admin
  Username: Your admin username
  Password: Your admin password

Step 2: Navigate to Scanner API Keys Tab
  Dashboard opens
  └─ Click on "Scanner API Keys" tab (top right of content area)
  └─ Next to "Registration Management"

Step 3: Generate New API Key
  Form appears: "Generate New API Key"
  └─ Enter Key Name: e.g., "Gate 1", "Registration Desk"
  └─ Click "Generate API Key" button

Step 4: Copy the Generated Key
  Success message appears with the generated key
  └─ Format: scanner_xxxxxxxxxx_xxxxxxxxxxxx
  └─ Click "Copy Key" button
  └─ Key copied to clipboard
  └─ WARNING: Can't see this key again, so save it!

Step 5: Share with Device Users
  Give the key to users who need to set up scanning devices
  └─ Users paste into mobile app's "API Key Setup" screen
  └─ App saves the key automatically
  └─ Ready to scan QR codes


───────────────────────────────────────────────────────────────────────────────
MANAGING API KEYS
───────────────────────────────────────────────────────────────────────────────

View All API Keys:
  In "Scanner API Keys" tab
  Shows a list of all generated keys with:
  └─ Key Name: Human-readable name
  └─ Status: Active/Inactive
  └─ Created: Date and time when generated
  └─ Last Used: When the key was used for a scan
  └─ Deactivate button: To revoke access

Deactivate a Key:
  Click "Deactivate" button next to the key
  Confirmation dialog appears
  └─ Confirm to deactivate
  └─ Key becomes inactive
  └─ Devices using this key will fail on next request
  └─ Users must get a new API key to continue

Track Device Usage:
  "Last Used" timestamp shows:
  └─ When the key was last used for a scan
  └─ Never: If key has never been used
  └─ Useful for auditing which devices are active

───────────────────────────────────────────────────────────────────────────────
CODE EXAMPLE: GENERATING API KEY VIA UI
───────────────────────────────────────────────────────────────────────────────

What Happens When User Clicks "Generate API Key":

1. Component State
   newKeyName: "Main Gate"
   generating: false → true

2. API Call
   POST /api/admin/scanner/api-keys
   {
     "action": "generate",
     "name": "Main Gate"
   }

3. Backend Processing
   └─ Generates key: scanner_1713272400000_abc123def
   └─ Stores in database
   └─ Returns: { apiKey, name, success }

4. Frontend Response
   generatedKey: {
     key: "scanner_1713272400000_abc123def",
     name: "Main Gate"
   }
   generating: false

5. UI Update
   Success message appears
   Key displayed in copyable format
   List refreshed to show new key
   Form cleared for next key

BACKEND API INTEGRATION
───────────────────────────────────────────────────────────────────────────────

Endpoint: /api/admin/scanner/api-keys

GET - Fetch all API keys
├─ Authentication: Admin session required
├─ Response:
│  {
│    "success": true,
│    "keys": [
│      {
│        "id": "507f1f77bcf86cd799439011",
│        "name": "Main Gate",
│        "isActive": true,
│        "createdAt": "2026-04-16T12:00:00Z",
│        "lastUsedAt": "2026-04-16T14:30:00Z"
│      }
│    ]
│  }
└─ Error handling included

POST (action: "generate") - Create new API key
├─ Authentication: Admin session required
├─ Request body:
│  {
│    "action": "generate",
│    "name": "Key Name"
│  }
├─ Response:
│  {
│    "success": true,
│    "message": "API key generated successfully",
│    "apiKey": "scanner_1713272400000_abc123",
│    "name": "Key Name"
│  }
└─ Error handling included

POST (action: "deactivate") - Deactivate API key
├─ Authentication: Admin session required
├─ Request body:
│  {
│    "action": "deactivate",
│    "keyId": "507f1f77bcf86cd799439011"
│  }
├─ Response:
│  {
│    "success": true,
│    "message": "API key deactivated successfully"
│  }
└─ Error handling included
