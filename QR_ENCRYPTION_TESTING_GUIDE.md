## Testing Steps

### Step 1: Backend Testing

**File**: `test/qr-generator.test.ts`

```typescript
import {
  encodeWith3DigitASCII,
  decodeFrom3DigitASCII,
  generateQRForRegistration
} from '@/lib/server/qr-generator'

describe('QR Generator - 3-Digit ASCII Encoding', () => {
  
  describe('encodeWith3DigitASCII', () => {
    it('should encode text correctly', () => {
      const text = 'p'  // ASCII 112
      const encoded = encodeWith3DigitASCII(text)
      expect(encoded).toBe('448')  // 112 * 4 = 448
    })

    it('should handle multiple characters', () => {
      const text = 'ab'
      // 'a' = 97, 97*4 = 388
      // 'b' = 98, 98*4 = 392
      const encoded = encodeWith3DigitASCII(text)
      expect(encoded).toBe('388392')
    })

    it('should pad to 3 digits', () => {
      const text = 'A'  // ASCII 65
      const encoded = encodeWith3DigitASCII(text)
      expect(encoded).toBe('260')  // 65 * 4 = 260 (already 3 digits)
    })

    it('should encode underscore', () => {
      const text = '_'  // ASCII 95
      const encoded = encodeWith3DigitASCII(text)
      expect(encoded).toBe('380')  // 95 * 4 = 380
    })

    it('should encode digits', () => {
      const text = '0'  // ASCII 48
      const encoded = encodeWith3DigitASCII(text)
      expect(encoded).toBe('192')  // 48 * 4 = 192
    })
  })

  describe('decodeFrom3DigitASCII', () => {
    it('should decode correctly', () => {
      const encoded = '448'  // Represents 'p'
      const decoded = decodeFrom3DigitASCII(encoded)
      expect(decoded).toBe('p')
    })

    it('should handle multiple characters', () => {
      const encoded = '388392'  // Represents 'ab'
      const decoded = decodeFrom3DigitASCII(encoded)
      expect(decoded).toBe('ab')
    })
  })

  describe('Round-trip Encoding/Decoding', () => {
    it('should encode and decode back to original', () => {
      const original = 'participant_solesta_SOL26-TEST01'
      const encoded = encodeWith3DigitASCII(original)
      const decoded = decodeFrom3DigitASCII(encoded)
      expect(decoded).toBe(original)
    })

    it('should handle special characters', () => {
      const original = 'participant_solesta_SOL26-ABC123_XYZ'
      const encoded = encodeWith3DigitASCII(original)
      const decoded = decodeFrom3DigitASCII(encoded)
      expect(decoded).toBe(original)
    })
  })

  describe('generateQRForRegistration', () => {
    it('should generate valid QR data', async () => {
      const result = await generateQRForRegistration('SOL26-TEST01')
      
      expect(result).toHaveProperty('qrCode')
      expect(result).toHaveProperty('encodedData')
      expect(result).toHaveProperty('rawData')
      expect(result.rawData).toBe('participant_solesta_SOL26-TEST01')
    })

    it('should generate valid base64 PNG', async () => {
      const result = await generateQRForRegistration('SOL26-TEST01')
      
      expect(result.qrCode).toMatch(/^data:image\/png;base64/)
    })

    it('should throw on missing referenceId', async () => {
      await expect(generateQRForRegistration('')).rejects.toThrow()
    })
  })
})
```

### Step 2: App Integration Testing

**File**: `App/services.test.ts`

```typescript
import { QRService } from '@/services'
import { decryptQRData } from '@/crypto'

describe('App QR Parsing', () => {
  
  describe('decryptQRData', () => {
    it('should parse backend-generated QR codes', () => {
      // This is what the backend generates (3-digit ASCII encoded)
      // We need to simulate what the scanner reads
      const backendEncoded = '414142434445464748494A4B4C4D4E4F505152535455565758595A2E5F' // participant_solesta_SOL26-
      
      const result = decryptQRData(backendEncoded)
      
      expect(result.isValid).toBe(true)
      expect(result.qrType).toBe('participant')
      expect(result.transactionId).toBeDefined()
    })
  })

  describe('QRService.parseQRData', () => {
    it('should parse valid encoded QR data', () => {
      const encodedQR = '414142434445464748494A4B4C4D4E4F505152535455565758595A' // participant_solesta_SOL26-
      
      const parsed = QRService.parseQRData(encodedQR)
      
      expect(parsed).not.toBeNull()
      expect(parsed?.type).toBe('participant')
      expect(parsed?.transactionId).toBeDefined()
    })

    it('should return null for invalid QR codes', () => {
      const invalidQR = 'invalid_qr_data'
      
      const parsed = QRService.parseQRData(invalidQR)
      
      expect(parsed).toBeNull()
    })
  })

  describe('QRService.validateQRData', () => {
    it('should validate parsed QR data', () => {
      const users = [
        {
          id: 'user1',
          name: 'John Doe',
          transactionId: 'SOL26-TEST01',
          email: 'john@example.com'
        }
      ]

      const qrData = {
        type: 'participant' as const,
        transactionId: 'SOL26-TEST01',
        timestamp: Date.now()
      }

      const validation = QRService.validateQRData(qrData, users)

      expect(validation.isValid).toBe(true)
      expect(validation.user?.name).toBe('John Doe')
    })
  })
})
```

## Known Issues & Resolutions

### Issue 1: Old QR Codes Still Use AES Encryption

**Problem**: Existing QR codes generated before this fix won't work

**Solution**: 
- Regenerate all QR codes after deployment
- Add backward compatibility in app (optional fallback)

### Issue 2: Encoding Length Mismatch

**Problem**: 3-digit ASCII encoding can create very long strings

**Example**: 
- Input: "participant_solesta_SOL26-TEST01" (32 chars)
- Output: 32 × 3 = 96 digits minimum

**Solution**: QR code error correction level H handles this

### Issue 3: Character Encoding Issues

**Problem**: Non-ASCII characters might not encode correctly

**Solution**: Stick to ASCII characters in reference IDs
- Allowed: A-Z, 0-9, `-`, `_`
- Avoid: Unicode, special chars (except `-` and `_`)

## Performance Considerations

### Encoding Speed
- Typical: <1ms for reference ID
- Acceptable: Fast enough for real-time QR generation

### QR Code Size
- 1 character in format = 3 digits
- "participant_solesta_SOL26-TEST01" = 96 digits ≈ 15-20KB PNG

### Network Transfer
- PNG image: ~5-10KB (base64 encoded)
- Acceptable for mobile networks

## Security Assessment

### Current Approach: 3-Digit ASCII Encoding

**Security Level**: Low (not cryptographic)
- Easily reversible
- No key needed
- Suitable for: Event QR codes (physical verification)

**Why it's okay**:
1. QR is displayed at entry point
2. Server validates against database anyway
3. No sensitive data in QR itself
4. Physical barrier prevents attacks

**When to upgrade**:
- If QR codes need to be sent remotely
- If sensitive data needs encoding
- If QR codes are reusable

## Support & Troubleshooting

### "QR code not found in registration data"

**Cause**: App can't decode the QR

**Fix**:
1. Verify backend is using new encoding
2. Regenerate QR codes
3. Clear app cache and resync
4. Check if old AES-encrypted QRs are still in use

### "Invalid encoded data format"

**Cause**: QR data doesn't follow 3-digit format

**Fix**:
1. Ensure backend generates correct format
2. Verify QR was scanned completely
3. Try with newer test QR code

### "Validation failed"

**Cause**: Transaction ID doesn't match database

**Fix**:
1. Verify registration exists
2. Check if fees are paid
3. Resync registration data
4. Verify reference ID format
